This is the primary language for building a webpage.

It makes the building blocks that will create a basic webpage.

To make a more pleasing or interactive website we need to use this in conjunction with other coding languages.

Media & Links

Tags

All of these tags can be used for external and local files.

Tag

Name

Description

Example (where needed)
<a> Link This causes the element/text contained to become a link. <a href="">Link.</a>
<img> Image This allows us to put an image into our webpage. It is a self closing tag so we only need the one tag. <img src="" alt="">
<video> Video This puts a video onto our page. This is not a self closing tag, so we should put some text between the tags which will displayif the video doesn't load. <video src="" width="" height="" controls>Video not supported</video>
<audio> Audio This tag is to seperate an audio file in our HTML. It does NOT contain any source for an audio file in the tag, but it can contain other attributes. <audio controls autoplay>.</audio>
<source> Source This is what we can use as a source for audio files, among others! It is self closing. <source src="" type="">
<embed> Embed This works like the video tag, but it can be used for many types of media, such as audio and .gif files. It is self closing.. <embed src="">
<link> Link We use this to link other documents to our page. This can be CSS or JavaScript documents so we can access them on our webpage. It is self closing. <link href="" rel="" type="">

Attributes

Attribute

Name

Description

Example (where needed)
href Hypertext Reference This is the URL for a link. It can reference an external link, a local file, or a position on the page. <a href="www.google.com">
<a href="./index">
src Source This is used to link media to our page, such as images, videos or audio. <embed src="./Resources/images/image1">
alt Alternate We use this with images. This will be text that appears if the image doesn't load, when the mouse hovers over it, or when someone uses a text to audio device. <img src="" alt="Picture of a dog">
width/height Width/Height We use this with videos to tell the browser the size of the video that we want in pixels. <<video src="" width="100px" height="100px" controls>
controls Controls We use this with video and audio to allow us to control the media (pause, play, scroll). <embed src="./Resources/images/image1">
autoplay Autoplay This causes the media to automatically play when the page loads. <audio controls autoplay>
type Type This informs the browser what type of file to expect. This could be telling the browser to expect an audio mp3, or a CSS text file <link href="" rel="" type="text/css">
rel Relation This explains to the browser how the link relates to the HTML file. <link href="" rel="stylesheet" type="">

Name

Description

Forms

Though we can create a form on our webpage with HTML, we need a different coding language to make it work properly! //add link here when I have worked it out!//

Tags

Tag

Name

Description

Example (where needed)
<form> Form This creates a form on the webpage for the user to fill in. <form action="" method="">>[all parts of it go here]</form>
<input> Input Here we add an input. The attributes specify what kind of input we want. We can also assign a default value that appears when it loads that users can override themselves. <input id="" type="" name="" value="">>
<label> Input Label This gives a label to the input. It must always go before the input and specify where it is for. The 'for' attribute must match the 'id' attribute. If we want it above the input, remember to add a br. <label for="">This is a label</label>
select Dropdown List This creates a dropdown list input. It must have the same id as the input element above it. <select id="lunch" name="lunch"></select>
datalist Datalist This creates a datalist. This is similar to a dropdown list (see select) but users can also input their own responses. This needs to be within an input element <datalist id=""></datalist>
option Options These are the options for your datalist or dropdown list for users to choose. They are child elements so must be wrapped in the tags for the parent element. <select id="lunch" name="lunch">
<option value="pizza">Pizza</option></select>
textarea Textbox This creates a text box for users to type an answer into. Like select and datalist it requires the same id as the input, but also needs its size specified. The user can expand the box if needed. <textarea id="" name="" rows="" cols=""></textarea>

Attributes

Attribute

Name

Description

Example (where needed)
action Action This is where the information needs to be sent. <form action="/example.html" method="">>
method Method This is the type of action being done. The method's value must be done in capital letters. <form action="" method="POST">>
type Type This is the type of input needed for the form. See the table below for more details. <input type="text" name="" id="">
name Name This is the name of the input. When submitted, this is paired with the input to help sort the data at the other end. For checkboxes and radio inputs, the name should always be the same. <input type="" name="food" id="">>
value Value This is what the prefilled value will be. <input type="text" name="first-text-field" value="already pre-filled">
for For We put this with the label to specify which input it is for. This MUST be the same as the id of the input. <label for="meal>Label title</label>
step Step This is for the arrows on a number input. It tells us how much to increase or decrease by when you click on the arrow. <input type="" name="" id="" step="2">
min/max Minimum/Maximum This is only used with range inputs (see below), and gives a minimum or maximum input. <input type="" name="" id="" value="" min="1" max="5">>
rows/cols Rows/Columns This is how we specify the default size for a textarea element. <textarea id="extra" name="extra" rows="3" cols="40"></textarea>

Input Type

Input Type

Name

Description

Example (where needed)
text Text This is a standard text input for short answers (such as a username). <input type="text" name="" id="">
password Password input THis acts like the text input but will replace the characters typed with a dot or *. This isn't secure though as it will still send the password in text form, so further security steps are needed. <input type="password" name="" id="">
number Number This only allows numbers in. It will also include arrows at the side of the input to increase or decrease the value. <input type="number" step="2" name="" id="">
range Range This creates aslider between a minimum and maximum value. Labels can be added to wither end. <span>Rare</span>
<input type="range" name="doneness" id="doneness" value="3" min="1" max="5">
<span>Well-Done</span>
checkbox Checkbox This creates a single checkbox which needs it's own label. For multiple checkboxes, this will need to be done lots of times. These should always be put in the same section so they stay together <input type="checkbox" name="" id="" value="">
radio Radio This creates a checkbox. The difference here is that the boxes are mutually exclusive. <input type="radio" name="" id="" value="">
submit Submit This is our submit button. The value attribute will give what is written on the button. <input type="submit" value="Submit">
